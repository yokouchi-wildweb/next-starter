# テンプレート利用ガイド

このリポジトリは汎用的な Next.js アプリ構築の土台として利用できます。サンプルとして投稿機能やメッセージ機能が含まれていますが、不要なドメインは自由に削除して構いません。

## 基本的な流れ

1. `features/` 以下に必要なドメインを追加します。既存の `event` などのフォルダは参考実装として残しています。
2. API ルートは `app/api/` 以下に配置し、REST 形式で命名します。
3. フロントエンド側の画面は `app/(user)/[route]` などに配置します。レイアウトやページ構成はプロジェクトに合わせて調整してください。

## カスタマイズのポイント

- ドメイン専用の型やサービスは `features/[domain]/` 内にまとめると管理しやすくなります。
- 共通UIコンポーネントは `components/` に追加してください。
- Firestore や Neon などの実装は `lib/` 配下に分離してあります。別のデータベースを利用する場合はここを差し替えてください。

このドキュメントに沿って構成をカスタマイズすることで、どのようなアプリにも転用できるテンプレートとして活用できます。

## 📁 ドキュメントフォルダ構成ガイドライン

このプロジェクトの開発者向けドキュメントは、
目的ベースで下記のフォルダに分類されています。

---

## `how-to/` - 実行手順・具体的なやり方

### 🔧 対象

- 「何をすればよいか」「どうやるのか」が知りたいとき

### 📘 例

- プロジェクトのセットアップ方法
- 開発・ビルド・デプロイ手順
- 特定機能の実装ガイド

---

## `reference/` - 仕様・リファレンス情報

### 📚 対象

- 「この関数はどう使う？」「この型の定義は？」といった調べ物

### 📘 例

- API仕様一覧
- コンポーネントのProps定義
- 環境変数や設定ファイルの項目説明

---

## `concepts/` - 設計思想・背景知識

### 🧠 対象

- 「なぜこうなっているのか」「どういう思想で設計されているのか」

### 📘 例

- アーキテクチャの概要
- 状態管理やルーティングの設計方針
- ライブラリや技術選定の理由

---

## `troubleshooting/` - トラブル対応・FAQ

### 🚨 対象

- 「エラーが出た」「原因不明の挙動がある」時に役立つ解決法

### 📘 例

- よくあるエラーとその解決策
- ビルド／デプロイ時のハマりポイント
- 開発環境特有の注意事項
