# テーブルコンポーネント利用ガイド

管理画面で共通的に利用できる `Table` コンポーネントの使い方をまとめます。

## 目的

- すべてのテーブルデザインを一箇所で管理する
- `variant` プロパティで簡単にスタイルを切り替えられるようにする

## 基本的な使い方

```tsx
import { Table, TableHeader, TableBody, TableRow, TableHead, TableCell } from "@/components/admin/table";

<Table variant="list">
  <TableHeader>
    <TableRow>
      <TableHead>ヘッダー</TableHead>
    </TableRow>
  </TableHeader>
  <TableBody>
    <TableRow>
      <TableCell>セル</TableCell>
    </TableRow>
  </TableBody>
</Table>;
```

`variant` は以下の2種類を用意しています。

| variant   | 説明                     |
| --------- | ------------------------ |
| `default` | Shadcn UI の標準スタイル |
| `list`    | 管理画面の一覧表示向け   |

## 切り替え方法

テーブルの見た目を変更したい場合は、画面側で `Table` コンポーネントに渡す `variant` を変更してください。新しいスタイルを追加したい場合は `src/components/admin/table/context.tsx` に variant を追記するだけで、管理画面内のすべてのテーブルに反映されます。

## AdminDataTable の利用

単純な一覧テーブルであれば、`AdminDataTable` を使うとカラム定義だけで表示を組み立てられます。

```tsx
import AdminDataTable, { AdminDataTableColumn } from "@/components/admin/AdminDataTable";

type Item = { id: string; name: string };

const columns: AdminDataTableColumn<Item>[] = [
  { header: "名称", render: (i) => i.name },
  {
    header: "操作",
    render: (i) => <a href={`/items/${i.id}/edit`}>編集</a>,
  },
];

<AdminDataTable items={items} columns={columns} getKey={(i) => i.id} />;
```

列ごとの `render` 関数でセル内容を自由にカスタマイズできます。
